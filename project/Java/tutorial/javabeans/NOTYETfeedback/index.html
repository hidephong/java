    The explanation for this is not given on this page, but on another page. Nevertheless, it definitely should be explained on this page too, even if briefly, because people are going to start wondering why this works.

  6. At ,in the section "Hooking Up Events in the BeanBox", it says that in the Juggler's EventTargetDialog,

    "This list contains Juggler methods that take no arguments, or arguments of type actionPerformed."

    "actionPerformed" should really be "ActionEvent".

    priority (1-5): 4

  7. At in the section "Property Editors", it says that

    "To support the custom property editor, you need to override two more methods: Override supportsCustomEditor to return true, and override getCustomEditor to return a custom editor instance. ColorEditor.getCustomEditor returns this."

    Not to be too picky, but I think we want "implement" rather than "override", since PropertyEditor is an interface.

    priority (1-5): 4

  8. In the section "How Property Editors are Associated with Properties", it says that

    "Property editors are discovered and associated with a given property by ... [e]xplicit association via a BeanInfo object. The Molecule demo Bean uses this technique. Within the MoleculeBeanInfo class, the Molecule Bean's property editor is set with the following line of code:

    pd.setPropertyEditorClass(MoleculeNameEditor.class);"

    Two things:

    1) We are setting the property editor for the Molecule Bean's "MoleculeName" property, not the MoleculeBean itself.

    2) What is "pd"? When you look at the code, it's a property descriptor. We should include the code that defines "pd" so that people know what is going on.

    priority (1-5): 4

  9. In the section "The BDK Property Editors", there is a reference to beans/apis/sun/beans/editors, which doesn't exist.

    priority (1-5): 4

  10. In the section "BDK Customizers", there is a reference to OurButtonCustomizer, which seems to have been replaced by ExplicitButtonCustomizer in the latest BDK.

    priority (1-5): 4

  11. Also, there is the snippet: public BeanDescriptor getBeanDescriptor() { return new BeanDescriptor(beanClass, customizerClass); } ... private final static Class customizerClass = OurButtonCustomizer.class;

    We should add the line that defines beanClass.

    priority (1-5): 4
